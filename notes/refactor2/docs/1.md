# 重构，第一个示例

1. 重构技术就是以微小的步伐修改程序，如果犯下了错误，你可以很容易修改它。
2. 重构后立即测试，即使出错了，也能把错误控制在很小的范围。
3. 重构后要提交代码，如果不小心出错了，也能轻松回滚到上一个可工作状态。在最后推送到远程仓库前，再把零碎的提交合并成一个有意义的提交。
4. 将函数的返回值命名为 result，因为看名字就能知道它的作用。
5. 变量命名是代码清晰的关键，只要改名能够提升代码的可读性，那就应该毫不犹豫地去做。
6. 如果重构引入了性能损耗，先完成重构，再做性能优化。
7. 把复杂的代码块分解为更小的单元，与好的命名一样都很重要。
8. 好代码的检验标准就是人们能否轻而易举地修改它。
